# 智餐优选 - 校园智能配送管理平台

## 项目简介

基于SpringBoot+MyBatis+Redis+MySQL构建的校园智能配送管理平台，采用微服务架构，实现了客户端和商家端的分离。客户端基于微信小程序支持用户在线下单、催单；商家端提供员工与商品管理、订单数据分析等功能。系统通过微服务架构提升了高并发场景下的可靠性和扩展性。

## 项目职责

- **用户认证与权限管理**：基于JWT令牌和微服务认证机制，实现分布式用户身份验证，通过ThreadLocal和拦截器处理Token验证，确保用户身份的安全。
- **缓存优化与安全设计**：使用Redis缓存高频数据，减轻数据库负载，提升系统响应速度，解决缓存穿透、雪崩和击穿问题。
- **多场景Redis应用**：通过Redis实现Session共享，确保用户登录状态在多个节点同步；使用Redis分布式锁和库存计数机制，防止超卖问题。
- **实时通知系统**：利用WebSocket和RabbitMQ进行订单通知和催单提醒，在客户端和商家端之间实现即时通信，保证订单状态实时同步。
- **分布式订单管理**：基于Spring Task处理订单的自动化管理，对超时订单实现定时取消。
- **服务发现与路由管理**：通过Nacos实现服务的注册与发现，并结合Spring Cloud Gateway进行请求路由和负载均衡。
- **分布式事务控制**：使用Seata确保跨服务操作的事务一致性。
- **全文检索功能**：通过Elasticsearch集成实现高效的数据查询。
- **流量监控与限流保护**：基于Sentinel实现各服务的流量监控、限流和熔断，确保系统稳定性。

## 技术栈

- **后端**：SpringBoot、SpringCloud（Gateway、Nacos、Feign、Sentinel）、MyBatis、Redis、MySQL、JWT、Seata、RabbitMQ、Elasticsearch
- **前端**：微信小程序

# 博客系统项目 - 基于 Golang 和 Vue 的全栈博客系统

## 项目简介

开发了一个基于Golang和Vue的全栈博客系统，支持多用户管理、文章发布、评论互动、用户权限控制等功能，现已上线。

访问地址：[www.hbbxht.top](http://www.hbbxht.top)
GitHub 地址：[GitHub Repository](https://github.com/wiseXumaster/gvb_server)

## 项目职责

- **后端开发**：基于 Golang 和 Gin 框架构建后端接口，完成博客内容管理和用户操作，使用 Gorm 进行 ORM 操作，确保 MySQL 数据库的高效访问。
- **前端开发**：使用 Vue.js 构建用户界面，设计响应式布局，实现前后端的数据交互和 UI 优化。
- **数据库设计与自动迁移**：设计并优化了 MySQL 表结构，使用 GORM 的 Automigrate 功能实现数据库的自动迁移。
- **命令行参数绑定与数据库管理**：使用 flag 包解析命令行参数，支持数据库迁移的动态控制。
- **Swag 集成与 API 文档生成**：集成 Swag 工具，自动生成接口文档，提高了 API 管理的规范性和团队协作效率。

## 技术栈

- **前端**：Vue.js
- **后端**：Golang、Gin、Gorm
- **数据库**：MySQL
- **其他**：JWT 认证、RESTful API、QQ 登录、七牛云图床、Swag

# 医疗知识图谱诊断系统

## 项目简介

开发了一个智能诊断系统，基于医疗知识图谱实现病例匹配和自动化诊断报告生成功能。

项目参考并集成了开源项目 [QASystemOnMedicalKG](https://github.com/liuhuanyong/QASystemOnMedicalKG) 构建了涵盖疾病、症状、药物等医疗实体及其关系的知识图谱。

## 项目职责

- **PyQt 开发与用户界面设计**：使用 PyQt 构建图形化用户界面，支持知识图谱查询、病例匹配和诊断报告查看。
- **嵌入浏览器展示图谱**：在 PyQt 界面中嵌入浏览器，用于展示知识图谱，为用户提供良好的视觉体验。
- **知识图谱可视化**：使用 pyecharts 进行知识图谱的可视化展示，帮助患者理解医学实体之间的关系。
- **病例影像展示**：支持 DICOM 和 NIfTI 格式医学影像文件展示，便于医生分析和比较病例数据。
- **传统匹配算法的应用**：在病例匹配中使用像素比对算法，基于影像的像素相似度进行匹配。
- **自动化诊断报告生成**：根据匹配的病例信息生成标准化诊断报告，自动填充患者信息、症状分析及治疗建议。

## 技术栈

- **界面开发**：PyQt，Qt 浏览器嵌入
- **知识图谱与问答系统**：基于 QASystemOnMedicalKG
- **数据处理**：Python、DICOM 解析库、NIfTI 支持、像素比对算法、NLP 工具
